214.最短回文串

问题

给定一个字符串 s，你可以通过在字符串前面添加字符将其转换为回文串。找到并返回可以用这种方式转换的最短回文串。

示例

示例 1：

示例 2：

提示

  • 0 <= s.length <= 5 * 104
  • s 仅由小写英文字母组成

解法一

思路

对于一个字符串s我们直接将其反转为reverse_s并将其添加到前面，显然reverse_s + s是一个回文串。但是他不一定是最短的回文串。这是因为当s前缀中存在回文串的时候，记作o，这样字符串s就能表达为：o + k。由于回文串的反转字符串依旧是它本身。

回文串：aba，反转之后：aba

我们可以得出reverse_s可以表达为：reverse_k + o。显然当前reverse_s + s是一个回文串但不是最短的回文串。这是因为reverse_s + s可以表达为：(revers_k + o) + (o + k)显然就算我们不添加o，reverse_k + o + k还是一个回文串，并且是最短回文串。

所以我们的题目就转换成找出字符串s的前缀子串中的最长回文串。

我们还可以将reverse_s添加到s的后面，这样就可以形成：(o + k) + (reverse_k + o)。我们要求的就是最长的字符串o，显然这是的o就是最长公共前后缀。所以我们可以使用KMP算法。

但是对于一种特殊情况却不行，那就是当字符串s本身就是一个回文串

s = “aaa”， reverse_s = “aaa”
s + reverse_S = “aaaaaa”其最长公共前后缀是6。但是我们的答案却是3

所以我们应该拼接为s + '*' + reverse_s这样就可以避免当s本身就是回文串的错误了。
